# テスト実行

pytestでテストを実行し、結果を分析するスキル。

## トリガー

ユーザーが「テスト」「pytest」を依頼した時、または実装完了後。

## 手順

1. テスト実行

```bash
pytest tests/ -v
```

2. 失敗がある場合は原因を分析
3. 修正案を提示

## テスト方針（Phase別）

| Phase | テスト対象 |
|--|--|
| 1 | データ整形、Pydanticスキーマ、JSON検証 |
| 1.5 | ガードレール（ルール違反検出） |
| 2 | SQLiteのCRUD、重複排除、紐付け |
| 3 | ベクトル検索精度、メタデータフィルタ |
| 4 | 既存テスト全通し + 条件分岐・ループ上限 |
| 5 | Webhook処理、通知変換 |

詳細は各docs/phase*.mdの「テスト方針」セクションを参照。
